# Git Commit 信息中文化规则

当生成 Git commit 信息时，请遵循以下规则：

## 语言要求

- **必须使用中文**描述提交内容
- 使用简洁、准确的中文表达
- 避免使用英文单词，除非是专业术语

## 提交信息格式

使用以下格式和对应的中文描述：

### 功能类型对照表

- `feat`: 新增功能
- `fix`: 修复问题
- `docs`: 更新文档
- `style`: 样式调整
- `refactor`: 重构代码
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建配置
- `ci`: CI配置
- `build`: 构建相关

### 示例格式

```
feat: 新增用户登录功能
fix: 修复路由跳转问题
docs: 更新项目README文档
style: 调整按钮样式和布局
refactor: 重构动态路由初始化逻辑
```

### 详细描述规则

- 主题行：简洁描述（50字以内）
- 如需详细说明，空一行后添加详细描述
- 使用现在时态："新增"而不是"新增了"
- 首字母无需大写
- 结尾不加句号

### 示例

```
feat: 新增动态路由配置功能

- 实现菜单数据扁平化处理
- 添加路由守卫确保动态路由初始化
- 支持全屏和布局内页面的路由配置
```
